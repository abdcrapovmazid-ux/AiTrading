AiTrading - Полная система регистрации

СТРУКТУРА ПРОЕКТА:
1. index.php - главная страница с формами входа и регистрации
2. register.php - обработчик регистрации
3. login.php - обработчик входа
4. dashboard.php - личный кабинет
5. logout.php - выход из системы
6. includes/config.php - настройки базы данных
7. includes/functions.php - вспомогательные функции
8. database/aitrading.sql - SQL скрипт для создания БД
9. css/style.css - стили приложения
10. js/script.js - JavaScript код

ИНСТРУКЦИЯ ПО УСТАНОВКЕ:

1. УСТАНОВКА НА ХОСТИНГЕ С PHP И MYSQL:

   a) Создайте базу данных на хостинге
   b) Импортируйте файл database/aitrading.sql в вашу БД
   c) Обновите настройки в includes/config.php:
      - DB_HOST (обычно localhost)
      - DB_NAME (имя вашей БД)
      - DB_USER (пользователь БД)
      - DB_PASS (пароль БД)
      - SITE_URL (адрес вашего сайта)

2. ЗАГРУЗКА ФАЙЛОВ НА ХОСТИНГ:

   a) Создайте папку на хостинге (например, aitrading)
   b) Загрузите все файлы и папки в эту директорию
   c) Убедитесь, что папки имеют права на запись (755)

3. НАСТРОЙКИ БЕЗОПАСНОСТИ:

   a) Измените права доступа к конфигурационным файлам
   b) На продакшене отключите вывод ошибок в config.php:
      ini_set('display_errors', 0);
      error_reporting(0);

4. ТЕСТИРОВАНИЕ:

   a) Откройте ваш сайт: вашдомен.xyz/aitrading/
   b) Зарегистрируйте тестового пользователя
   c) Используйте тестовые данные для входа:
      Email: test@example.com
      Пароль: Test123456

ВАЖНЫЕ ФУНКЦИОНАЛЬНОСТИ:

1. Система регистрации с валидацией:
   - Проверка email
   - Проверка пароля (мин. 6 символов)
   - Подтверждение пароля
   - Проверка уникальности email

2. Система входа:
   - Проверка учетных данных
   - Хэширование паролей
   - Запоминание сессии

3. Личный кабинет:
   - Отображение статистики
   - История сделок
   - Управление профилем
   - Подключение Telegram

4. Безопасность:
   - Хэширование паролей (password_hash)
   - Защита от SQL-инъекций (PDO prepared statements)
   - CSRF защита
   - Логирование активности

ДОПОЛНИТЕЛЬНЫЕ НАСТРОЙКИ:

1. Для работы с Telegram:
   - Создайте бота через @BotFather
   - Получите API токен
   - Настройте вебхук в настройках бота

2. Для работы с платежами:
   - Настройте API ключи платежных систем
   - Обновите таблицу payments в БД

3. Для работы с email:
   - Настройте SMTP сервер
   - Обновите функцию sendEmail в functions.php

ПРИМЕЧАНИЯ:

1. Тестовый пользователь уже добавлен в БД (пароль: Test123456)
2. Все пароли хранятся в хэшированном виде
3. API ключи генерируются автоматически при регистрации
4. Для продакшена обязательно:
   - Используйте HTTPS
   - Настройте безопасные куки
   - Регулярно делайте бэкапы БД

ПОДДЕРЖКА:

При возникновении проблем:
1. Проверьте права доступа к файлам
2. Проверьте настройки подключения к БД
3. Включите вывод ошибок в config.php для отладки
4. Проверьте логи веб-сервера
